# What's left to make TopRankr production-ready

## 1. Recommendation Engine

Currently you save:

```text
ranking_score = 91.2
```

Convert this into recruiter-friendly labels:

```text
95+   → Excellent Match
85+   → Strong Match
70+   → Good Match
50+   → Potential Match
<50   → Not Recommended
```

Example:

```json
{
  "candidate_id": 4,
  "ranking_score": 91,
  "recommendation": "Strong Match"
}
```

---

## 2. Save Final Results in SearchJob.result

Currently:

```text
SearchJob
    id
    status
    progress
    current_step
```

Store:

```json
{
  "top_candidates": [...]
}
```

This allows:

```http
GET /recruiter/jobs/5
```

to instantly return cached results.

---

## 3. Candidate Details API

Create:

```http
GET /candidate/{candidate_id}
```

Return:

```json
{
    "id": 4,
    "name": "...",
    "email": "...",
    "phone": "...",
    "profile": {...},
    "summary": "...",
    "resume_path": "..."
}
```

---

## 4. Resume Download API

Instead of:

```json
{
    "resume_path": "uploads/resume.pdf"
}
```

implement:

```http
GET /candidate/4/download
```

which returns the actual PDF.

---

## 5. Recruiter Feedback Loop

Allow recruiters to mark:

```text
✓ shortlisted
✓ rejected
✓ interviewed
✓ hired
```

This becomes training data for future ranking improvements.

Example table:

```sql
recruiter_feedback
    id
    job_id
    candidate_id
    decision
```
